# Linux Utility Scripts Collection

这是一个旨在简化 Linux 系统管理、自动化运维任务和环境配置的脚本仓库。

---

## 🚀 项目初衷

在管理 Linux 服务器时，我们经常需要执行一些繁琐的初始化、监控或清理任务。这个项目的目标是：

* **一键化**：将复杂的命令组合成简单的一键执行脚本。
* **标准化**：提供统一的配置和优化方案。
* **轻量化**：脚本无过多依赖，开箱即用。

---

## 🛠️ 功能模块

本仓库包含以下几类脚本（根据你的仓库内容持续更新）：

| 脚本分类 | 说明 | 推荐场景 |
| --- | --- | --- |
| **系统优化** | 内核参数调优、BBR 开启、Swap 增加 | 新服务器初始化 |
| **环境安装** | Docker 等常用软件一键安装 | 快速部署开发环境 |
| **网络工具** | 测速脚本、防火墙配置 (iptables/ufw) | 网络排查与安全加固 |

---

## 📖 如何使用

### 1. 克隆仓库

首先，将整个项目克隆到本地：

```bash
git clone https://github.com/cxwya/linux_script.git
cd linux_script

```

### 2. 赋予执行权限

在使用任何脚本之前，请确保它具有可执行权限：

```bash
chmod +x <脚本名称>.sh

```

### 3. 运行脚本

通常建议使用 `sudo` 权限运行系统级脚本：

```bash
sudo ./<脚本名称>.sh

```

---

## ⚠️ 注意事项

* **备份先行**：在生产环境运行任何修改系统配置的脚本前，请务必做好数据备份。
* **权限说明**：大部分脚本涉及系统修改，需要 `root` 或 `sudo` 权限。
* **系统兼容性**：主要针对 **Ubuntu/Debian** 和 **CentOS** 系列进行测试，其它发行版请自行测试。
